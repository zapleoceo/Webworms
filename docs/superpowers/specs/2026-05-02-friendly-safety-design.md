## Цели

1) В конце хода бот не должен оставаться в позиции, где одним взрывом выгодно задеть сразу двух союзников.
2) При выборе выстрела бот должен учитывать риск задеть союзника по линии огня (hitscan или полёт снаряда) и избегать этого через штраф.
3) После выстрела projectile-оружием бот может сместиться, пока снаряд летит, чтобы уменьшить риск быть наказанным или попасть под собственный следующий шот.

## Термины

- projectile-оружие: оружие, создающее `state.projectiles` (bazooka, homing_missile, grenade, minigun и т.п. в текущих типах).
- hitscan: оружие, создающее взрыв через raycast без projectile.

## Входные данные и ограничения

- На стороне AI доступны позиции всех живых worm’ов (союзники/враги), их loadout (equipmentIds) и cooldowns.
- `Weapon.explosionRadius` используется как основной показатель “опасного радиуса”.
- `botCfg.scoring.safeExtraRadius` используется как дополнительный запас.
- Изменения должны быть устойчивыми: не ломать “спасение” в последние секунды хода.

## Изменение 1: Анти-скученность в конце хода (end-turn spacing)

### Расчёт радиуса угрозы

Вычислять два значения:

- `R_threat_enemy = max(explosionRadius доступного оружия у всех живых врагов) + safeExtraRadius + padding`
- `R_threat_self = max(explosionRadius доступного оружия у нашей команды) + safeExtraRadius + padding`

Итог:

- `R_cluster = max(R_threat_enemy, R_threat_self)`

Рекомендуемый `padding`: 18–26px (примерно размер worm’а/погрешности траекторий).

### Штраф при выборе moveTo

Если бот выбирает `moveTo`, в scoring позиции добавлять штраф:

- для каждого союзника `ally` (кроме самого shooter):  
  если `dist(moveTo, ally) < R_cluster`, добавить штраф, растущий по мере приближения.

### Условие применения

- Жёстко применять только если есть время на reposition: `turnTimeLeft > reserveSeconds + 3s`.
- Если времени мало: применять мягко (меньший коэффициент), чтобы не ломать атаку/спасение.

## Изменение 2: Риск союзника по линии огня (friendly line risk)

### Геометрия

Для кандидата выстрела считать минимальную дистанцию от каждого союзника до линии огня:

- hitscan: сегмент `muzzle -> hitPoint`
- projectile: сегмент `muzzle -> predictedImpact` (из симуляции траектории / res.end)

Если минимальная дистанция меньше порога `R_line`, добавлять штраф:

- `R_line = ally.width/2 + safeExtraRadius` (или фиксированный минимум 24–30px)
- штраф должен быть сильным, но не абсолютным запретом (по требованию).

## Изменение 3: Reposition после выстрела projectile (post-shot reposition)

### Условие включения

Если:

- `hasFiredThisTurn === true`
- `state.projectiles.length > 0`
- и `turnTimeLeft > 0` и есть запас времени (например `turnTimeLeft > 1.2s`)

Тогда бот может выполнять перемещение, пока есть projectile.

### Ограничения

- Никаких повторных выстрелов (`fire` запрещён).
- Никаких rope attach в этом режиме.
- Цель перемещения: увеличить расстояние до союзников относительно `R_cluster` и/или выйти из “плохой” зоны (минимизировать суммарный spacing-штраф).

## Критерии успеха

- Количество кейсов “два союзника стоят в радиусе взрыва” после завершения хода заметно падает.
- Количество friendly-fire / почти-friendly-fire по линии выстрела снижается.
- В projectile-ходах бот иногда делает шаг/прыжок после выстрела, если это уменьшает риск, и не ломает логику завершения хода.
