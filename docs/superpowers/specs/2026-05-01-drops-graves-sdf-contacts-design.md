# Дропы и могилки: SDF-контакты как у гранаты

## Цель
- Применить к дропам (BrandLogo пока `isDynamic=true`) и могилкам (grave BrandLogo) тот же принцип взаимодействия с террейном, что и у гранаты: **signed distance + нормаль** вместо перебора пикселей/многочисленных коллизий.
- Оставить текущее поведение после стабилизации: превращение в текстуру через `Landscape.stampImage(...)`.
- Починить отображение спрайта могилы (сейчас виден fallback).

## Текущее состояние
- Граната: использует `TerrainDistanceField` для `signedDistance()` и `normal()` и решает контакт импульсами в `PhysicsEngine.updateGrenadeProjectile`.
- Дропы/могилки: `BrandLogo.update` вызывает `integrateAirdrop` ([AirdropPhysics.ts](file:///workspace/src/physics/AirdropPhysics.ts)), где контакты строятся по набору точек прямоугольника и террейн проверяется через `solid(...)` + дискретная оценка нормали по соседям + пошаговое “выпихивание” (pen loop).
- Рендер могилы: могилки — это BrandLogo со спрайтом `/sprites/Misc/graveN.png`; если картинка не загрузилась до момента отрисовки/штампа — виден fallback.

## Решение (рекомендованное)
### 1) SDF для контактов прямоугольника
В `integrateAirdrop`:
- Подключить `TerrainDistanceField` и кэшировать его per Landscape (WeakMap).
- Для каждой контактной точки `wp` вместо `solid(ix,iy)`:
  - `sd = df.signedDistance(wp.x, wp.y)`
  - если `sd >= 0` — контакта нет
  - если `sd < 0` — есть проникновение `pen = clamp(-sd, 0, cfg.maxPenetration)`
  - нормаль `n = df.normal(wp.x, wp.y)`
- Penetration correction и импульсы оставить в текущем стиле AirdropPhysics (restitution + friction + solverIterations), но использовать `pen` и `n` из SDF.

Ожидаемый эффект:
- заметно меньше “коллизий” и стабильнее поведение на наклонных поверхностях,
- меньше CPU (нет step-by-step push-out по пикселям).

### 2) Могилки
Могилки уже используют BrandLogo + integrateAirdrop, значит автоматически получают SDF-контакты.

### 3) Отображение спрайта могилы
Добавить предзагрузку `/sprites/Misc/grave1..6.png` в `CanvasRenderer` (wormImages cache), чтобы:
- при падении не было fallback,
- при стабилизации `stampImage` гарантированно имел загруженный src для штампа.

## Критерии готовности
- Дропы и могилки при падении/отскоках ведут себя заметно “гладче” (без дрожания/залипания) на неровностях и наклонах.
- После стабилизации продолжают штамповаться в текстуры как раньше.
- Могилка рисуется корректным спрайтом, fallback не появляется.
