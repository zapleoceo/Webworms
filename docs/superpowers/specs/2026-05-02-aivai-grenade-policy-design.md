# AIVAI: Grenade Policy (Ammo-aware) + Safe Reserve Fire

## Контекст

В AIVAI у гранаты есть лимит 15 на команду за матч (`teamAmmo.teamX.grenade`), и выстрел гранатой уменьшает лимит. Сейчас ИИ может:

- тратить гранаты в `reserve_fire` (fallback);
- делать “положить гранату под себя/своих” (self/FF), что одновременно ухудшает качество игры и сжигает дефицитный ресурс.

## Цели

- Граната становится оружием “только если выгодно”: высокий expectedDamage при низком self/FF risk.
- Резервное действие (`reserve_fire`) не может по умолчанию выбирать гранату и стрелять опасно.
- Алгоритм остаётся предсказуемым и поддаётся тюнингу через параметры (в botCfg или константы).

## Не-цели

- Не делаем полный тактический планировщик “позиция → движение → серия выстрелов”.
- Не гарантируем, что бот всегда стреляет каждый ход.

## Принятое допущение

Если нет безопасного/выгодного выстрела, бот может завершить ход без выстрела (лучше “не стрелять”, чем self/FF + потеря дефицитной гранаты). Если это окажется нежелательно, можно вернуть “обязательный выстрел” отдельным флагом.

## Изменения в выборе оружия (BotAI)

### 1) Scarcity penalty (штраф за дефицит гранат)

В `chooseBotActionScored(...)` уже читается `grenLeft` из `world.teamAmmo[team].grenade`. Добавить к скорингу кандидата гранаты штраф, растущий при уменьшении `grenLeft`.

Пример функции штрафа:

- `scarcity = 1 / (grenLeft + 1)`
- `score -= grenadeScarcityWeight * scarcity`

Параметры:

- `grenadeScarcityWeight` (по умолчанию 20–60, подстройка по логам)

Интерпретация:

- пока гранат много (15 → 10) штраф небольшой;
- когда гранат мало (3 → 0) граната почти всегда проигрывает альтернативам, если она не даёт большой expectedDamage.

### 2) Hard safety gates для гранаты

Даже если score высокий, граната отбрасывается при выполнении одного из условий:

- angle почти вертикальный и дистанция маленькая (эвристика против “положить под себя”);
- predicted impact слишком близко к стрелку;
- predicted friendly/self damage выше порога.

Параметры:

- `grenadeMinSelfDistancePx` (например 90–120)
- `grenadeMaxAbsAngleForClose` (например 1.2 рад при close-range)
- `grenadeMaxFriendlyDamage` (например 6–10 ожидаемого урона)

## Резервное действие (BotTurnController)

### 3) Safe reserve fire (без “grenade by default”)

Сейчас `fallbackAction()` выбирает grenade, если она есть. Это корректно для “демо”, но некорректно для AIVAI.

Предложение:

- В `reserve_fire` вызывать общий выбор выстрела (тот же, что используем для execute) с жёсткими порогами безопасности.
- Если нет кандидата, который проходит thresholds — не стрелять.

Пороговый режим `reserve`:

- `expectedDamage >= reserveMinExpectedDamage`
- `friendlyDamage <= reserveMaxFriendlyDamage`
- `selfDamage <= reserveMaxSelfDamage`

Если выстрел не найден:

- бот завершает ход без выстрела.

## Тестирование

- Unit: если `teamAmmo.grenade=0`, BotAI никогда не выбирает grenade.
- Unit: если кандидат grenade имеет predicted self/FF риск, он отбрасывается даже при хорошем expectedDamage.
- Integration: reserve_fire не выбирает grenade по умолчанию; при отсутствии safe shot не стреляет.
