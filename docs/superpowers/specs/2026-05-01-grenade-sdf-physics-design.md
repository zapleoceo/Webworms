# Реалистичная физика гранаты (SDF + импульсы)

## Цель
- Убрать текущий “дорогой” перебор коллизий гранаты (сотни сабшагов × проверки террейна по множеству оффсетов).
- Сделать поведение гранаты ближе к реальной физике: парабола полёта, отскоки от наклонных поверхностей, естественное скольжение/скатывание, стабилизация в покое.
- Обеспечить одинаковое поведение гранаты во всех режимах (training/ai/friend/random) и на всех клиентах.

## Текущее состояние (как сейчас)
- Интеграция гранаты и коллизии выполняются в [PhysicsEngine.updateProjectile](file:///workspace/src/presenters/PhysicsEngine.ts).
- Анти-туннелинг сделан через sweep и сабстеппинг до 360 шагов на тик; на каждом сабшаге проверяется террейн по ring offsets, а после удара ещё делается цикл “выпихивания”.
- Отскок реализован отражением скорости по нормали + трение, но нормаль берётся дискретно и не хватает механики “реального” контакта с угловой скоростью (spin), что ухудшает скатывание/устойчивость.

## Предлагаемый подход (Variant A)
Ввести для террейна Signed Distance Field (SDF) и перевести гранату на импульсную физику контакта (как rigid body “круг”), используя SDF для:
- быстрого определения минимального расстояния до поверхности,
- получения нормали поверхности через градиент,
- продвижения по траектории крупными шагами без многократного перебора пикселей.

### Ключевая идея
Вместо “проверяем много пикселей вокруг гранаты”:
- читаем `dist = SDF(x, y)` в центре гранаты,
- определяем зазор `sep = dist - r`,
- если `sep > 0`, можно двигаться вперёд на большой шаг,
- если `sep <= 0`, это контакт: делаем penetration correction вдоль нормали и считаем импульсы (restitution + трение + spin).

Ожидаемая сложность: обычно 1–3 итерации на тик, вместо `maxSubsteps × steps × ringOffsets`.

Подход обновления:

- Пересчитывать SDF локально в окрестности dirty-region с запасом `margin = rGrenade + 20`.
- Пересчёт локального SDF делать “multisource” по границе солид/воздух внутри окна:
  - строим очередь из всех “границ” (air cell, рядом solid) и/или (solid cell, рядом air),
  - распространяем расстояния по 8 направлениям (дёшево и стабильно),
  - нормаль берём из локального градиента по `dist` (центральная разность) и квантируем.

Цель: корректность и стабильность для гранаты; абсолютная математическая точность EDT не обязательна, важнее гладкая нормаль и отсутствие “проваливания”.

## Новая физика гранаты

### Состояние гранаты
Расширить `GrenadeProjectile`:
- `radius` уже есть (6).
- добавить `mass`, `inertia`, `angularVelocity` уже есть.
- добавить `restitution`, `dynamicFriction`, `staticFriction`, `rollingResistance`.
- добавить режим `sleep/resting` (уже есть) с более физичным критерием.

### Интеграция (per fixed step)
1) Силы:
- `vy += gravity * dt`
- `vx += wind * dt * windMultiplier`
2) Предсказание перемещения:
- `dx = vx * dt`, `dy = vy * dt`
3) TOI-цикл (до N итераций, например 4):
- читаем `dist = SDF(x, y)`, `n = normal(x, y)`
- `sep = dist - r`
- если `sep > 0`:
  - разрешаем шаг `t = min(1, 0.9 * sep / max(eps, |v|*dt))`
  - `x += dx * t`, `y += dy * t`
  - уменьшаем оставшийся `dtRem`
- иначе (контакт):
  - penetration correction: `x += n.x * (-sep + slop)`, `y += n.y * (-sep + slop)`
  - считаем контакт в точке `p = center - n * r`
  - решаем импульсы:
    - `vContact = v + ω × rVec`
    - `vn = dot(vContact, n)`
    - если `vn < 0`, применяем нормальный импульс с restitution
    - вычисляем касательную `t`, применяем тангенциальный импульс с кулоновским ограничением `|jt| <= μ * jn`
  - дополнительное “rolling resistance” гасит ω и тангенциальную скорость при устойчивом контакте.

### Скатывание
Скатывание получается естественно как:
- компонента гравитации вдоль касательной на наклонной поверхности,
- отсутствие отрыва при малой нормальной скорости,
- статическое трение предотвращает микроскольжение при малых скоростях.

### Покой (sleep)
Ввести критерий:
- если `|v| < vSleep` и `|ω| < wSleep` в течение `sleepTime`, то `resting=true`.
- при `resting=true` граната перестаёт двигаться (до нового импульса от взрыва/толчка).

## Детерминизм
- Использовать fixed timestep (уже есть в PhysicsEngine).
- Нормаль SDF хранить квантованной (Int8), чтобы минимизировать платформенные расхождения Float.
- Избегать `Math.random`/недетерминированных источников в гранатной физике.

## План тестирования
- Добавить тесты на сценарии:
  - отскок от горизонтали: угол падения == угол отражения с учётом restitution,
  - отскок от наклона: скорость после удара меняет направление ожидаемо,
