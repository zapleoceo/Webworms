## Цель

1) Сделать симуляцию траектории выстрела у бота максимально близкой к реальной физике игры, чтобы на hard “идеальные” выстрелы не промахивались из‑за расхождения моделей.
2) В админке добавить массовый апскейл всех custom maps до высоты 1000px с сохранением пропорций и перезаписью текущих карт (updateMap на тот же id).
3) Снизить загрузку CPU без изменения геймплея (семантика столкновений/урон/траектории сохраняются).

## Контекст

- Реальная физика снарядов использует substepping (≈ 1 шаг на пиксель траектории) и проверку коллизий с террейном по радиусу, плюс коллизии с игроками/пропами/brand logos: [PhysicsEngine.updateProjectile](file:///workspace/src/presenters/PhysicsEngine.ts#L696-L883).
- AI симуляция в [PhysicsHelper.simulateTrajectory](file:///workspace/src/ai/PhysicsHelper.ts#L34-L89) упрощена: фиксированный dt (1/60), проверка одной точки без радиуса и без substepping, поэтому возможен “tunneling” на высокой скорости.
- Апскейл карт уже есть как операция “Upscale 4K” в админке, но цель — унификация высоты до 1000px и массовая обработка.

## Дизайн

### 1) Shared симуляция траектории (Game + AI)

**Новая утилита:** `src/physics/TrajectorySim.ts` (или аналогичный существующий модуль в physics-слое).

**Функции:**
- `simulateProjectilePath(terrain, start, velocity, gravity, wind, windMultiplier, radius, maxTime, dtBase)`:
  - substepping по длине сегмента: шаги = `ceil(distancePerFrame)` (как в game-physics), минимум 1;
  - проверка террейна по радиусу: вместо двойного цикла по квадрату использовать предрасчитанные оффсеты круга для `pr`;
  - возвращает `end`, `minDistToTarget`, `hitTerrain`.
- `simulateGrenadePath(...)`:
  - сохраняет текущую модель bounce/restitution/friction/stopSpeed, но использует тот же substepping и проверку террейна по радиусу.

**Интеграция:**
- Game: `PhysicsEngine.updateProjectile` оставляет текущую механику, но выносит “горячие” проверки террейна/сабстеппинг в переиспользуемые функции (см. оптимизацию ниже).
- AI: `simulateTrajectory` переиспользует общий симулятор, чтобы совпадали туннелинг/радиус/ветер.

**Инварианты:**
- Входы gravity/wind/windMultiplier совпадают с игровыми.
- Порядок приоритета коллизий сохраняется (entity до terrain), если остаётся необходимость точного соответствия.

### 2) Admin: массовый апскейл custom maps до 1000h (overwrite)

**UI:**
- В секции Maps добавить кнопку “Upscale ALL to 1000h”.
- Показывать прогресс: `processed/total`, текущую карту, и возможность отмены (toggle флаг).

**Алгоритм:**
- Для каждой карты:
  - загрузить `/api/maps/:id` чтобы получить ссылку на изображение;
  - загрузить PNG (CORS уже настроен);
  - посчитать `targetH = 1000`, `targetW = round(origW * targetH / origH)`;
  - нарисовать в `canvas` размером `targetW x targetH` с `imageSmoothingEnabled=true` и сохранить как `image/png` (`toDataURL('image/png')`), прозрачность сохраняется;
  - вызвать `PUT /api/admin/maps/:id` с `image_data`, `width=targetW`, `height=targetH`, и обновлённым `name` (обновить `(WxH)` суффикс).

**Ограничения:**
- Сохранить текущий лимит 25MB на image_data на backend (если карта не вписывается — показать ошибку и продолжить следующую).

### 3) CPU-оптимизация (без изменения геймплея)

**Цели оптимизации:**
- Снизить стоимость `updateProjectile` (самый горячий участок) и минимизировать количество тяжёлых вложенных циклов на каждый substep.

**Изменения:**
- Предрасчитать оффсеты круга для террейн‑проверки по радиусу (для типичных `pr` 1..5) и использовать их вместо квадратного двойного цикла.
- Broadphase для сущностей при substepping:
  - перед циклом substeps собрать список кандидатов (players/props/logos), чьи AABB пересекаются с swept-AABB сегмента (old->new расширенный на radius);
  - внутри substep перебирать только кандидатов.
- Уменьшить аллокации в hot loop (избегать `filter`/создания массивов каждый кадр там, где можно).

**Что НЕ меняем:**
- Формулы урона/ветра/гравитации.
- Порядок обработки событий (взрыв, bounce, fuse).

## Диагностика и валидация

- Unit: текущие `npm test` должны проходить.
- Ручная проверка:
  - hard AI: выстрелы не “улетают” из‑за несоответствия симулятора и реальной физики (особенно на высоких скоростях).
  - карты после апскейла корректно загружаются и имеют размеры `*x1000`.
- Performance:
  - сравнить FPS/CPU до и после на одинаковой карте (воспроизводимый сценарий: много снарядов/бренд‑логотипов/пропов).

## Риски

- При полном совпадении физики AI может стать сильнее (особенно hard). Это ожидаемо и соответствует “pro”.
- Массовый апскейл увеличит размер grid в Landscape (1000h) → может поднять CPU на рендер/физику. Оптимизации projectile collision должны компенсировать часть нагрузки, но при необходимости потребуется отдельная оптимизация загрузки/рендера террейна.
