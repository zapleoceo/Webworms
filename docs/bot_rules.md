## Правило проекта

Любое изменение поведения бота (логика выбора цели/позиции/оружия, тайминги, использование верёвки, правила безопасности, логика гранаты) требует обновления этого файла.

## Дизайн стратегии

Полное описание новой стратегии: [bot_strategy_mcts.md](./bot_strategy_mcts.md).

## Тайм-бюджет хода

- `planSeconds`: сколько секунд бот планирует (в это время не выполняет действий).
- `reserveSeconds`: сколько секунд в конце хода резервируется под “хотя бы один выстрел”.
- `executionSeconds = turnTime - planSeconds - reserveSeconds`.

## Обзор алгоритма (на ход бота)

1) Планирование выполняется в WebWorker.
2) План строится через MCTS:
   - действия абстрагированы на уровне хода: `MoveTo(x)` и `Shot(...)`,
   - непрерывные параметры (угол/сила) расширяются progressive widening,
   - часть rollout’ов включает “ответ противника” (2‑ply) смешанно: по триггерам и по бюджету.
3) Исполнение:
   - бот двигается к `moveTo`,
   - затем выполняет `action` (выстрел),
   - если осталось ≤ `reserveSeconds` и бот ещё не стрелял — выполняет резервный выстрел.

## Правила безопасности

- Self‑damage запрещён (hard‑filter).
- Friendly‑damage:
  - easy/medium: допускается, но штрафуется (soft penalty),
  - hard: максимально избегается (жёсткий фильтр или большой штраф).

## Сложность (через глубину и бюджет)

Сложность задаётся профилем бюджета поиска:
- количество итераций MCTS,
- maxDepth / rolloutDepth,
- размер action space (topMoves/topShots),
- частота 2‑ply.

Шум в прицеливании допускается как дополнительный “человеческий фактор”, но не является главным механизмом различия сложности.

## Верёвка (ограничения)

Верёвка используется только для достижения позиции перед выстрелом.

- Лимит считается по количеству успешных attach за ход:
  - easy: ropeAttachLimit.easy
  - medium: ropeAttachLimit.medium
  - hard: ropeAttachLimit.hard

## Верёвка (использование)

- Перед попыткой attach бот проверяет, что по лучу прицеливания есть точка зацепа (raycast), чтобы не тратить попытки впустую.
- Бот выбирает угол attach из нескольких кандидатных углов и использует тот, который действительно цепляется и даёт лучший потенциал перемещения.
- Верёвка используется как набор стратегий: rope_climb / rope_swing / rope_descend (подбирается по контексту цели).
- Во время движения на верёвке бот оценивает прогресс по упрощённой стоимости пути (cost) и отцепляется при устойчивой стагнации (не по одному неудачному “тику”).
- Для rope_swing бот старается отпускать крюк позже (по таймингу/скорости), чтобы долетать дальше, а не падать раньше времени.

## Сброс верёвки в конце хода

Если червь заканчивает ход с активной верёвкой, верёвка принудительно отключается при переходе хода.

## Режим Dig (копать под движение)

Если бот не смог найти валидный план выстрела/позиции, он может сделать “копающий” выстрел рядом с собой, чтобы изменить рельеф и сделать следующую позицию достижимой.

- Dig включается параметром dig.enabled.
- Максимум копающих выстрелов за ход: dig.maxShotsPerTurn (обычно 1).
- Бот выбирает точку копания в локальной зоне по направлению к ближайшему врагу:
  - X = shooter.x ± dig.distances[]
  - Y = topSolidY(X) + depth (между dig.depthMin..dig.depthMax)
- Далее подбирает выстрел разрушительным оружием, который максимально близко попадает в точку копания и не задевает себя/союзников.

## Прыжок (перемещение)

При перемещении к позиции бот пытается перепрыгивать препятствия:
- если впереди в направлении движения обнаружен материал (простая проверка “стенка/выступ”) — бот отдаёт приоритет прыжку,
- если бот “застрял” (позиция X почти не меняется некоторое время) — бот выполняет прыжок.

## Перепланирование и анти-зацикливание

- Бот выбирает стратегию перемещения (walk/jump/rope*) по контексту и статистике успешности в рамках матча.
- Если стратегия не даёт прогресса, бот переключается на другую.
- Если стратегия проваливается или выбирается более 3 раз за ход — она запрещается до конца хода.
- При накоплении провалов по нескольким стратегиям бот сбрасывает текущий план и строит новый (replan).

## Параметры (admin → BOT)

- planSeconds
- reserveSeconds
- ropeAttachLimit.easy / medium / hard
- grenade.fuseSeconds / restitution / friction / stopSpeed
- scoring.killBonus / damageWeight / missWeight / movePenaltyPerPx / safeExtraRadius
- dig.enabled / dig.maxShotsPerTurn / dig.distances / dig.depthMin / dig.depthMax
- movement.maxStrategyAttemptsPerTurn / maxStrategyFailuresPerTurn / replanWhenBannedAtLeast / replanCooldownSeconds

## Спавн (безопасность)

- Точки спауна валидируются точной проверкой AABB (коробка червя + clearance должна быть полностью в воздухе).
- Y спауна привязан к поверхности: выбирается пиксель поверхности (solid с воздухом над ним), и червь ставится над ним.
