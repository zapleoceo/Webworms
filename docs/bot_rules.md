## Правило проекта

Любое изменение поведения бота (логика выбора цели/позиции/оружия, тайминги, использование верёвки, правила безопасности, логика гранаты) требует обновления этого файла.

## Тайм-бюджет хода (30 секунд)

- Bot Plan Seconds: сколько секунд бот “думает” и не выполняет действий.
- Bot Reserve Seconds: сколько секунд оставляется в конце хода на резервный выстрел.
- Execution Seconds = Turn Time - Plan Seconds - Reserve Seconds.

Во время полёта/взрыва снаряда таймер хода продолжает тикать.

## Цель и приоритеты

- Главная цель: нанести максимальный ожидаемый урон по врагу.
- Приоритет: если есть шанс добить врага одним выстрелом (по оценке ожидаемого урона), такой вариант получает дополнительный бонус.
- Безопасность: варианты, которые могут задеть бота или союзников взрывом, отбрасываются.

## Обзор алгоритма (на ход бота)

1) Если сейчас идёт планирование (elapsed < planSeconds) — бот ничего не делает.
2) По окончании планирования строится план:
   - выбираются кандидаты позиций (пешком + расширение по дальности с учётом возможности верёвки),
   - для каждой позиции перебираются варианты выстрела (оружие/угол/сила),
   - выбирается план с лучшим скорингом минус штраф за перемещение.
3) На этапе исполнения бот перемещается к целевой позиции:
   - пешком,
   - либо используя верёвку (полный контроль), но с лимитом attach по сложности.
4) После достижения позиции (или исчерпания execution window) бот выполняет выстрел.
5) Если времени осталось ≤ reserveSeconds и бот ещё не стрелял — выполняется резервный выстрел из текущей позиции.
6) После выстрела ход заканчивается, когда в мире больше нет снарядов.

## Выбор выстрела (перебор и скоринг)

Для каждого живого врага:

- перебираются доступные оружия из экипировки (weapon equipment без cooldown),
- перебираются углы и сила,
- симулируется траектория:
  - обычные снаряды: симуляция до столкновения с террейном/лимита времени,
  - граната: симуляция до fuseSeconds с упрощёнными отскоками/скольжением, оценка по позиции взрыва в t=fuseSeconds,
- вычисляется:
  - miss: расстояние от точки взрыва/ближайшей точки траектории до цели,
  - expectedDamage: weapon.damage с линейным спадом по explosionRadius,
  - kill: expectedDamage ≥ health цели,
- score = expectedDamage * damageWeight - miss * missWeight + (kill ? killBonus : 0),
- вариант отбрасывается, если точка взрыва ближе (explosionRadius + safeExtraRadius) к боту или любому союзнику.

## Сложность (только через погрешность выстрела)

Планирование и перебор одинаковые на всех сложностях. Разница только в финальной погрешности перед выстрелом:

- easy: aimErrorPct и powerErrorPct (например ±30%)
- medium: (например ±15%)
- hard: (например ±5%)

## Верёвка (ограничения)

Верёвка используется только для достижения позиции перед выстрелом.

- Лимит считается по количеству успешных attach за ход:
  - easy: ropeAttachLimit.easy
  - medium: ropeAttachLimit.medium
  - hard: ropeAttachLimit.hard

## Параметры (admin → BOT)

- planSeconds
- reserveSeconds
- ropeAttachLimit.easy / medium / hard
- aimErrorPct.easy / medium / hard
- powerErrorPct.easy / medium / hard
- grenade.fuseSeconds / restitution / friction / stopSpeed
- scoring.killBonus / damageWeight / missWeight / movePenaltyPerPx / safeExtraRadius
